# GitHub 上传工具

自动检测问题并上传到 GitHub 的脚本工具。

## 文件说明

- `github-push.ps1` - PowerShell 主脚本（功能完整）
- `push.bat` - 批处理文件（简化调用）

## 使用方法

### 方法一：使用批处理文件（推荐）

```cmd
push.bat "提交信息"
```

示例：
```cmd
push.bat "feat: 添加新功能"
push.bat "fix: 修复 bug"
push.bat "docs: 更新文档"
```

### 方法二：使用 PowerShell 脚本

```powershell
.\github-push.ps1 -Message "提交信息"
```

可选参数：
- `-Branch main` - 指定分支（默认: main）
- `-Force` - 强制推送
- `-NoProxy` - 禁用代理

示例：
```powershell
.\github-push.ps1 -Message "feat: 新功能" -Branch main
.\github-push.ps1 -Message "hotfix: 紧急修复" -Force
```

## 自动检测功能

脚本会自动检测以下问题：

### ✅ 基础检查
- [x] Git 仓库是否存在
- [x] Git 用户配置是否完整
- [x] 工作区是否有未提交更改

### 🌐 网络检查
- [x] GitHub 直连是否可用
- [x] 代理是否可用（V2RayN / Clash）
- [x] 自动配置代理端口

### 📦 上传检查
- [x] 远程分支是否存在
- [x] 是否有冲突需要解决
- [x] 推送权限是否正常

## 代理支持

脚本会自动检测以下代理软件：

| 软件 | 端口 |
|------|------|
| V2RayN | 10808, 10809, 10810 |
| Clash | 7890, 7891 |

检测到代理后会自动配置，上传完成后自动清除代理配置。

## 错误处理

如果上传失败，脚本会：
1. 显示具体的错误信息
2. 提供解决建议
3. 保持代理配置（如果启用）

## 首次使用

1. 确保已安装 Git
2. 配置 Git 用户信息：
   ```cmd
   git config user.name "你的名字"
   git config user.email "你的邮箱"
   ```
3. 确保有网络连接或代理可用
